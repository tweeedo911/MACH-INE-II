## Sanit├а тАФ Check eseguiti

### 1. Baseline regression (default state = no-op)

**Stati nuovi con default no-op in `world-state.js`:**
- `rootOffset: 0` тЖТ `root + 0` = nessuna transposizione
- `densityMultiplier: 1.0` тЖТ `if (dMul !== 1.0)` skippa il loop
- `meloMuteBars/bassMuteBars: 0` тЖТ rami `if (> 0)` non entrano
- `ritornoVariant: 'default'` тЖТ branch `isTempestaToRitorno` cade nel flow standard
- `preSuiteActive: false` тЖТ `updateDirector3` early-return disattivato

**Risultato:** senza trigger (no hotkey, no `?presuite`, no `?seed`), il comportamento
├и bit-equivalente a v3.17.1. Verificato via grep su tutti i callsite.

### 2. Hot path sanit├а

- `campo.js` render loop: nessuna allocazione nuova. `tCos/tSin` pre-calcolati 1├Ч per
  frame fuori dal loop cella. `BAYER4N`, `_geoMemory`, `_geoMemoryRGB` allocati una
  sola volta in `_ensureBuffers()`. Omen loop pixel-wise opera in-place su `data`.
- `director3.js` update: nessuna closure nuova per tick. `for (const mod of [...])`
  itera array letterale, OK (V8 lo solleva). Nessun `new` in tick.
- `main.js` loop: nessun `new` in loop principale. HUD flash usa `setTimeout` gi├а
  esistente.
- `audio.js/midi.js`: ring buffer rimuove `.shift()` O(n). `_recentOnsets()` alloca
  un Array di max 40 elementi solo quando BPM effettivamente cambia (raro).
  **Nota:** allocazione in hot path di audio, ma тЙд40 elem e rate <1Hz, trascurabile.

### 3. Conflitti naming / import

- Tutti gli import risolvono. Export di director3 verificati: `setRitornoVariant`,
  `startPreSuite`, `endPreSuite`, `resetDramaturgyState` presenti alle righe
  230/238/251/265.
- `clearAll as clearCampo` importato corretto da `campo.js`.
- `SeededRNG` esportato da `perf-utils.js`.
- Nessun identifier shadow rilevato (grep su nuovi nomi: uso confinato).

### 4. Safety nets attivi

- **Shift+Z panic**: chiama `sendMIDIAllNotesOff`, `clearCampo`, azzera `density`,
  `resetDramaturgyState` (incluso pre-suite). Ogni step in try/catch indipendente.
  HUD flash 2s. Ripristino stato HUD warnings dopo flash.
- **`document.hidden` guard**: early-return in `loop()`, re-schedule rAF. Listener
  `focus/visibilitychange` riprende AudioContext se suspended.
- **Ring buffers**: `_pushNoteTimestamp`/`_pushOnsetTimestamp` usano modulo power-of-two.
  `_countNoteTimestampsAfter` walk al pi├╣ count entries (O(cap)). Zero `.shift()`.

### 5. Feature correctness

- **A1 rupture hooks**: optional chaining `worldState?.rupture?.stage`. `rupture` ├и
  sempre inizializzato in world-state (non null), ma la cautela ├и safe.
- **A2 humanize**: `track?.humanize?.velocity ?? fallback` in ogni call site. Fallback
  pari al valore hard-coded precedente (6, 4, 8, VOICE_HUMANIZE).
- **B1 geologia**: `clearAll()` chiama `_geoMemory.fill(0)` + `_geoMemoryRGB.fill(0)`.
  Verificato. Alloc only in `_ensureBuffers` (1├Ч/resize).
- **B2 MACCHINA grid**: `if (density < 0.5) continue` skippa intera cella, **corretto**.
- **B2 TEMPESTA vector**: `tCos/tSin` 1├Ч/frame, `proj = (px*tCos + py*tSin)|0` solo
  math integer. No alloc.
- **B2 NEBBIA radial**: `else if (isNebbiaRadial && role === 'voice')` тАФ condizione
  role-specific corretta.
- **B3 omen lerp**: `(r*204 + (255-r)*51) >> 8` = `(153r + 13005) >> 8`. Max 52020 /
  256 = 203. No overflow (int32 safe). Coefficienti sommano a 255 = full range.
- **C1 phrygianHold**: `initDirector3('RITORNO')` aggiorna `_track` a RITORNO; poi
  override `scale`/`root` con TEMPESTA. `_track.bpm` ├и RITORNO.bpm (60). Ramp
  `TEMPESTAтЖТRITORNO` bar standard. Variant reset a default (one-shot).
- **C1 silenceThenAeolian**: `_silencePending` flag + accumulator. Density azzerata
  ogni tick durante silenzio. Dopo 90s: init RITORNO normale.
- **C3 octave clamp**: `Math.max(-24, ...)` e `Math.min(24, ...)`. Corretto.
- **C3 density clamp**: `Math.min(2.0, ...)` e `Math.max(0.3, ...)`. Corretto.
- **C4 sub drone**: nota `root-24` parallela al drone (`rootOct`). Non ├и stessa nota
  тЖТ nessuna double-note collision su CH2. Safety `subNote >= 12` previene note <C0.

## Issue rilevate

### Minor (non bloccanti)

1. **Mute bar decrement al cambio fase** (`director3.js:528-531`):
   `_lastMuteBarChecked` pu├▓ differire da `_phaseBars=0` al nuovo phase, causando
   un decremento extra all'entrata nella nuova fase. Effetto pratico: mute si chiude
   1 bar prima del previsto quando attraversa confine di fase. Accettabile, il
   performer pu├▓ ri-triggerare M/N se serve.

2. **`_recentOnsets()` alloca `new Array(cap)`** (`audio.js`): in hot path di BPM,
   ma cap тЙд40 e chiamato solo quando un onset effettivamente supera `bpmMinInterval`
   (rate bassa). GC pressure trascurabile.

3. **Optional chaining su `worldState.rupture` superfluo**: `rupture` ├и sempre
   inizializzato. Tuttavia ├и una cautela difensiva, nessun impatto perf.

### Warning (richiedono monitoring live)

4. **B3 omen fullscreen pass**: `~0.2ms/frame` stimato ma non misurato live.
   Monitorare in TEMPESTA dove omen pu├▓ durare a lungo.

5. **Sub drone tattile su NEBBIA/TESSUTO**: vel 20-30 su C1 (~32Hz). Se l'output MIDI
   finisce in un synth con filter low-shelf aggressivo, rischio di note inudibili ma
   che occupano voices. Testare col set-up live prima.

6. **Pre-suite drone loop**: `setTimeout` interno a `_preSuiteDroneAcc` ├и basato su
   `dt` del clock tick (~2ms). Ogni 4s emette note-on su CH2 con durata 4s. OK ma
   se `dt` drifta sotto pressione, note si sovrapporranno (la successiva emette
   prima che la precedente si sia spenta). Accettabile: drone aumenta leggermente
   la sonorit├а, non crea dissonanza.

### Nessun Blocker

- Nessuna regressione baseline.
- Nessun breakage sintassi (`node --check` pulito su tutti i 33 file .js).
- Nessun import mancante.
- Nessun conflitto naming.
- Safety nets tutte attive.
