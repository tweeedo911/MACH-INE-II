**Sezione: Gating e MIDI optimization wrapper**
- Funzione `sendNote(ch, note, vel, dur)`:
  - Gating: `if (macroState.rhythmicDensity < 0.05 && ch !== CFG.RHYTHM.midi.channels.hat) return;`
  - **Eccezione hi-hat (RITM-01, D-06):** CH1 bypassa il gating di rhythmicDensity — usa `velFloor` come minimo assoluto. Il hat manda sempre, la velocity e' clampata al floor.
  - Pitch range enforcement (MIDI-02): `note = Math.max(range.min, Math.min(range.max, note))`
  - Downbeat boost (MIDI-01): se `_currentStep16 % 4 === 0` aggiungi `vel *= (1 + CFG.RHYTHM.midi.downbeatBoost)`, altrimenti `vel *= (1 - CFG.RHYTHM.midi.offbeatReduce)`
  - Humanize: `vel += (Math.random() - 0.5) * CFG.RHYTHM.kick.humanizeRange` (o il range del canale)
  - Clamp: `vel = Math.max(1, Math.min(127, Math.round(vel)))`
  - Phrase offset (MIDI-03): `const offset = Math.random() * (CFG.RHYTHM.midi.noteOffsetMs.max - CFG.RHYTHM.midi.noteOffsetMs.min) + CFG.RHYTHM.midi.noteOffsetMs.min;` — usare `setTimeout(() => _rawSend(...), offset)` per sfasamento microtemporale (nota: setTimeout e' disponibile nel main thread, il worker posta messaggi al main; tuttavia nel contesto attuale siamo nel callback del worker che gira sul main thread, quindi setTimeout funziona)

- Funzione dedicata `sendHatNote(note, vel, dur)`: identica a sendNote ma senza gating rhythmicDensity — applica solo velFloor, pitch range CH1, downbeat/offbeat, humanize.

**Sezione: CH0 Kick — onKickStep(step)**
- Step sequencer a 16th-note come in composer.js
- `const s16 = step % 16;`

- Nella fase `arhythmic`: non fare nulla (D-03)
- Nella fase `emerging`: probability gate — `if (Math.random() > CFG.RHYTHM.kick.gateProbability.emerging) return;` — poi selezionare random tra `broken1`, `broken2`, `broken3` e suonare se `pattern[s16]` (D-02: frammentato ma non casuale)

- Nella fase `dissolving`: mirror di emerging, probability gate `dissolving`, pattern broken1

**Sezione: CH1 Hi-hat — onHatStep(mainStep)**
- **Due clock indipendenti** per phasing Reich (D-11, RITM-04):
  - Clock A avanza con step length `CFG.RHYTHM.hat.phasingStepsA` (8)
  - Clock B avanza con step length `CFG.RHYTHM.hat.phasingStepsB` (9)
  - Entrambi usano lo stesso BPM ma il pattern A completa un ciclo in 8 step e B in 9, creando drift naturale
  - Convergenza ogni `8 * 9 = 72` step (~8-10 secondi a 88 BPM)
- Nella fase `arhythmic` e `dissolving`: solo clock A (no phasing, D-12), step divisor da CFG (ogni 8th note)
- Nelle fasi `emerging`, `groove`, `climax`: entrambi i clock attivi
- Per ogni hit:
  - Pitch rotation (D-04): `const pitch = CFG.RHYTHM.hat.pitchCluster[_hatPitchIdx % CFG.RHYTHM.hat.pitchCluster.length]; _hatPitchIdx++;`
  - Velocity scatter gaussiano (D-05): `let vel = velTarget[phase] + gaussianRandom() * CFG.RHYTHM.hat.velScatter;` — dove `gaussianRandom()` e' approssimata con `(Math.random() + Math.random() + Math.random() - 1.5) * 2` (distribuzione approssimata, media 0)

**Sezione: Init e Update exports**
```javascript
export function initRhythmLayer() {
  _clock = 0; _lastStep = -1; _bar = 0;
  _currentPhase = 'arhythmic';
  _kickPatIdx = 0;
  _hatClockA = 0; _hatLastStepA = -1;
  _hatClockB = 0; _hatLastStepB = -1;
  _hatPitchIdx = 0;
  _percActiveNotes = new Set();
  _percPatterns = {}; _percClocks = {}; _percLastSteps = {};
  _specialFired = { falseRes: false, climax: false, dissolve: false };
  _prevRhythmicDensity = 0;
  sendMIDIAllNotesOff();
  console.log('[RHYTHM] init');
}

export function updateRhythmLayer(dt) {
  _updatePhase();
  _updatePercAdditive();  // check additive entry Glass

  // Step sequencer principale — 16th note @ bpmReference
  const bpm = CFG.RHYTHM.bpmSource === 'macro' ? CFG.MACRO.bpmReference : CFG.RHYTHM.bpm;
  const stepsPerSec = bpm * 4 / 60;
  _clock += dt * stepsPerSec;
  const currentStep = Math.floor(_clock);
  if (currentStep > _lastStep) {
    for (let s = _lastStep + 1; s <= currentStep; s++) {
      _onMainStep(s);
    }
    _lastStep = currentStep;
  }

  // Hi-hat phasing clocks (indipendenti dal main step per drift)
  _updateHatPhasing(dt, bpm);

  // Percussion sub-clocks (lunghezze prime)
  _updatePercClocks(dt, bpm);

  // Special events check
  _checkSpecialEvents();
}
```

- `_onMainStep(step)`: chiama `_onKickStep(step)` per CH0. Il hi-hat e le percussioni hanno clock separati.
- `_updateHatPhasing(dt, bpm)`: avanza `_hatClockA` e `_hatClockB` indipendentemente, fire `_onHatHit()` quando crossano un nuovo step nel loro ciclo
- `_updatePercClocks(dt, bpm)`: per ogni nota in `_percActiveNotes`, avanza il clock e fire hit quando crossano uno step nel pattern
