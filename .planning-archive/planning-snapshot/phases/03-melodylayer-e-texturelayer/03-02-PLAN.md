**Wrapper gating `sendNote(ch, note, vel, dur)`** — pattern harmony-layer.js/rhythm-layer.js:
- Gate su `macroState.melodicActivity < CFG.MELODY.activityGateFloor` => return (no emit)
- Pitch clamp a `CFG.MELODY.pitchRange[ch].min` / `.max`
- Velocity scalata per `macroState.melodicActivity`
- Downbeat boost e offbeat reduce via `CFG.MELODY.midi.downbeatBoost` / `.offbeatReduce`
- Humanize gaussiano: `vel += _gaussianRand() * CFG.MELODY.velHumanize`
- Clamp velocity: `Math.max(1, Math.min(127, Math.round(vel)))`
- MIDI-03 offset anti-meccanico: `setTimeout(() => _rawSend(ch, note, vel, dur), offset)` con offset da `CFG.MELODY.midi.noteOffsetMs`
- Log: nessun console.log per ogni nota (solo per seed events)

**Weighted Markov `_nextVoiceNote(ch, history, seedIntervals)`** — derivato da `composer.js` `nextVoiceNote()`:
- Pool di note dalla scala corrente: `CFG.MACRO.modes[macroState.currentMode]`
- Filtrare le note nel pitch range del canale: `CFG.MELODY.pitchRange[ch]`
- Peso per ogni nota:
  - Base: `w = 1.0`
  - Se `history[1] !== null` (nota precedente esiste):
    - `diff = n - history[1]`
    - Se `Math.abs(diff) <= 3`: `w *= CFG.MELODY.markov.stepBonus` (2.5 default)
    - Se `Math.abs(diff) > 7`: `w *= CFG.MELODY.markov.jumpPenalty` (0.15 default)
    - Per CH6 (differenziazione D-03): applicare `ch6JumpPreference` (1.6) come moltiplicatore aggiuntivo per salti > 3 semitoni, e `ch6StepReduction` (0.7) come riduttore del stepBonus
    - Per ogni intervallo in `seedIntervals`:
      - Se `mi === diff`: `w *= CFG.MELODY.markov.seedAffinity` (1.8)
      - Else se `Math.abs(mi - diff) === 1`: `w *= CFG.MELODY.markov.seedNearAffinity` (1.3)
- Weighted random selection: `total = sum(w)`, `r = Math.random() * total`, iterate fino a `r <= 0`
- Aggiornare history: `history[0] = history[1]; history[1] = nota_scelta`
- Ritornare la nota scelta

**Step handler `_onCH3Step(stepInLoop)`** — bassline melodica (D-04):
- Determinare fase melodica da `macroState.melodicActivity`: < 0.3 => 'sparse', < 0.65 => 'medium', >= 0.65 => 'dense'
- Probabilistic gating: `Math.random() < CFG.MELODY.emitProbability.ch3Base * macroState.melodicActivity` — se false, skip (silenzio su questo step)
- Per la fase 'sparse' usare una soglia piu bassa di gating (`ch3Base * 0.5 * mA`) per frasi piu rade
- Nota via `_nextVoiceNote(3, _histCH3, _seedMotif || [])` — CH3 usa la propria history Markov (`_histCH3` dichiarata in stato interno)
- Velocity da `CFG.MELODY.velTarget[fase].ch3 + _gaussianRand() * CFG.MELODY.velHumanize`
- Durata da `CFG.MELODY.noteDur.ch3[fase === 'dense' ? 'dense' : 'sparse']`
- `sendNote(3, note, vel, dur)`

**Step handler `_onCH5Step(stepInLoop)`** — voce melodica con seed (D-02, D-07):
- Probabilistic gating: `Math.random() < CFG.MELODY.emitProbability.ch5Base * macroState.melodicActivity`
- Eccezione per seed capture: se `!_seedCaptured && macroState.arcPercent < CFG.MELODY.seedWindowEnd`, il gating minimo diventa `CFG.MELODY.seedCaptureGateMin` (0.02) invece di `activityGateFloor` — permette alle prime note di uscire per catturare il seed
- Nota via `_nextVoiceNote(5, _histCH5, _seedMotif || [])`
- Velocity e durata come CH3 ma con valori CH5
- `sendNote(5, note, vel, dur)`
- **Seed capture** (dopo sendNote): incrementare `_ch5PhraseNoteCount`. Se `_ch5PhraseNoteCount >= 2` e `_histCH5[0] !== null`, calcolare intervallo `_histCH5[1] - _histCH5[0]` e pusharlo su `_intervCH5`. Quando `_intervCH5.length >= CFG.MELODY.seedLength` e `!_seedCaptured` e `macroState.arcPercent < CFG.MELODY.seedWindowEnd`: `_seedMotif = _intervCH5.slice(0, CFG.MELODY.seedLength); _seedCaptured = true; console.log('[MELODY] seed captured:', _seedMotif);`

**Step handler `_onCH6Step(stepInLoop)`** — voce indipendente (D-03):
- Probabilistic gating: `Math.random() < CFG.MELODY.emitProbability.ch6Base * macroState.melodicActivity`

**Seed return `_checkSeedReturn()`** (D-08, D-09):
- Guard: `if (_seedReturned || !_seedCaptured || macroState.arcPercent <= CFG.MELODY.seedReturnAt) return;`
- `_seedReturned = true;`
- `console.log('[MELODY] seed returned at arcPercent:', macroState.arcPercent);`
- Calcolare root: `const rootNote = CFG.MACRO.droneRoot[macroState.currentMode] || 57;`
- Enqueue le note del seed nel micro-sequencer `_seedReturnQueue`:
  ```javascript
  let note = rootNote;
  let targetBar = macroState.barClock;
  const scale = CFG.MACRO.modes[macroState.currentMode] || [];
  for (const interval of _seedMotif) {
    _seedReturnQueue.push({ note, vel: CFG.MELODY.seedReturnVel, dur: 800, targetBar });
    const candidate = note + interval;
    // Snap a nearest scale note per coerenza modale
    note = scale.reduce((best, n) =>
      Math.abs(n - candidate) < Math.abs(best - candidate) ? n : best, scale[0]);
    targetBar += CFG.MELODY.seedReturnNoteSpacingBars;
  }
  ```
- **NON usare setTimeout** — il Worker non lo supporta in modo affidabile. Usare il micro-sequencer basato su barClock.

**Seed return flush `_flushSeedQueue()`**:

- Nota: usa `_rawSend` diretto (bypassa gating — il ritorno del seed DEVE suonare indipendentemente da melodicActivity)
